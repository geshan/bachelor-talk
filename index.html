And how you can use your bachelor study to craft your tech career

  • What are its future prospects?
  • Software Engineer, Web Developer, Sys Admin, Business Analyst, Product Mgmt

  • Could be same in Nepal if you work at Verisk or Yomari
  • Full of challenges and you are paid good even from starting.
  • Compare starting with at bank 6-8k to 10-12k.

  • Could be a similar situation in Nepal
  • It's very hard to find good programmers/software engineers here as well.

    Things to consider when you select and do projects for bachelors

    • Not a hard and fast checklist, still great to have there 7-8 things.
    • MoScow Model - http://en.wikipedia.org/wiki/MoSCoW_method

    Do a marketable product as a project which has utility, and can be sold in future

    Not each time Library management system, stock management or HR management
    • Rent Management System
    • Building Management System
    • Appogee Leave Management
    • RentACoder.com
    • HireACoder.com
    • Appvita.com
    • Producthunt.com
    • Leanstack.io/categories
    Keep your eyes open for API integration. Try online payment sandbox like Paypal.

    How about Rent/Building Management System?

  • Each building has ~12 floors, each floor has ~10 flats
  • Each building has 24 hour security
  • Each security makes a security round every 1/2 hours and writes report
  • Each building has paid/free maintenance
  • Each building has visitor log

    • Back To Kathmandu, Nepal. A foreign employment management system.

  • Each day thousands (~1915 per day in 2070) go to foreign countries for jobs - http://www.dofe.gov.np/uploads/document/Chaitra%202070_20140430052407.pdf
  • Around 800 of foreign employment agencies exist. - http://www.nafea.org.np/members/page/26

    • Always use a project management system/tool, you can start with Trello or Redmine

    Using a source code version control system is a necessity, I prefer Git which is DVCS. Use bitbucket for hosting.

    Write more specs and code than words on the report

  • Writing clear specs is very important but only writing specs is not good.

    • For web projects, think in terms of APIs than user interface, consider REST with JSON. Think Mobile, try Ionic.

    Do a JSON Rest Webservice API on 4th sem and do a Android mobile app on 5th Java

    Weekly update to academic supervisor or lecturer.

    Even presentation can be HTML, Javascript and CSS, like Reveal.js and host it

    Continuity : If you contribute 1 hour per weekend for 2 years it's 100 hours for your project. Be consistent, focus.

    • College needs a new website with CMS. My Success story not migrated to Wordpress
    • Sir, do you need any software now?

    Barter your working project with scholarship.
    Why should scholarships always be given on academic performance?

    • Example of X college Y student with 20% scholarship for merit and system dev.

    Secrets of selecting a company and doing internship

    Start early.

    Try finding a company that will host you as in intern by end of 6th sem.

    So start searching for one from 5th sem, surely not 7th sem.

    Select a medium sized organization, not too big and not too small - like Young Innovations Private Limited (YIPL)

    • If its too small nothing to do and learn, if too big too bureaucratic, no info about company and you might end up just photocopying.

    College should assign academic supervisor and assessor for internship

    • BSC IT curriculum mentions it.
    • Mentor for great results.

    Do requirement specs very well. If you can, follow agile methodology.

    Do not reinvent the wheel.
    Use Open Source software

    Depending on the project, give priority to using a framework or a Content Management System (CMS).

    • PHP - Symfony, Zend, CodeIgniter, Laravel
    • PHP - Drupal, Joomla, Wordpress

    4 years, one of the longest time you study. Use it well.

    How to approach the job market.

    Make a CV/resume that accentuates your skills

    • Reverse chronological
    • Do not repeat things
    • Mention college projects, host them.

    Focus on one language/framework/CMS (have idea of the others).
    Don't become jack of all master of none.

    Start writing technical blog posts

    • Appreciated in job interviews

    Network and search a job with commitment

    • LinkedIn.com
    • JobsNepal.com
    • MeroJob.com
    • Jagire.com
    • RamroJob.com

    Be prepared for coding challenges.
    Interviews will be there, coding challenges will help you seal the deal.

    • Good skill and thinking test

    Continuous Learning, technology changes fast.

    Conclusion/Recap

    • Be informed of the latest technology, framework and other news in the tech/web dev industry.
    • Start a blog, make a Linked In profile. Have some good GitHub public activity.
    • Bachelor study is 4 years, utilize it to the fullest.
    • Passing exams is a must, just being able to program will not earn you a degree.
    • Use academic projects to build a ladder to a great internship project.
    • Exploit your internship opportunity to land your first job.
    • Focus on a language or framework or CMS, don't try to learn everything.
    • Learning is a continuous process even after you get a job.
    • Suggestion : Do a Master after working for 2-3 years.
